## 들어가며
'객체 지향 프로그래밍', OOP란 단어를 처음 들어본 시점 자체는 사실 컴퓨터 학원에서 C++을 배우던 중학생 때였다. 하지만 고등학교와 대학교 시절 내내 Python과 C++을 배우면서도, 이 객체 지향 프로그래밍이라는 패러다임이 대체 무엇을 의미하는지, 기초적인 개념조차 제대로 이해하지 못하고 있었다. 이는 고등학교와 대학교 교양강의 수준에서 하는 프로그래밍이 사실 기초 문법 수준에서 벗어나지 않았고, 거대한 프로젝트를 만드는것이 아닌 문제풀이 알고리즘 짜기에서 벗어나지 않았기에 절차 지향적 프로그래밍만으로 충분했기 때문이다.

하지만 이젠 단순 알고리즘 풀이가 아니라 실제 프로젝트를 진행한다. 프로젝트가 커지고 기능이 많아질수록 전역 네임스페이스 문제가 점점 커질 것이며, 프로젝트의 결과물은 더이상 중고등학교 때 만들던 'CLI 환경에서 입력을 받으면 연산을 하고 출력하면 끝'인 프로그램이 아니다. 그래서 객체 지향 프로그래밍의 패러다임을 이젠 이해해야 한다. 

이제야 객체 지향 문법을 배우는 단계이기에, 어떠한 경우에 객체 지향적으로 구현해야 하는지, 객체 지향 프로그래밍의 원칙(SOLID)이 무엇인지 등의 관념적인 것들은 해당 포스트에서 다루지 않을 것이다. 이번 TIL 포스트에서는 JS에서의 객체 지향 프로그래밍을 위한 문법과 키워드들만을 간단히 정리해보려 한다. 


## Today I Learned
### 1. `constructor` 함수 (생성자 함수)
`constructor`는 이전 <a href="https://victorybh04.github.io/posts/TIL-2025-11-13-this-%EB%B0%94%EC%9D%B8%EB%94%A9/" target="_blank">`this` 바인딩에 관한 포스트</a>에서 간단하게 다룬 적이 있다. `constructor`는 'construct'라는 단어의 '건설하다'라는 의미에서 알 수 있듯, 무언가(객체)를 생성한다는 뜻에서 '생성자'라고도 부른다. 

**생성자 함수는 같은(비슷한) 구조를 가지는 객체를 여러 개 생성하고 싶을때 사용한다.** 즉 일종의 `Object` 복사 기계와 같다. `function` 키워드를 이용해 생성하며, 함수처럼 파라미터를 넣을 수 있다. 생성자 함수는 일반 함수와 구분하기 위해, 관습적으로 이름을 대문자로 시작하도록 짓는다.

```js
function Person(name, age){
    this.name = name;
    this.age = age;
    this.sayHi = function(){
        console.log(`Hello, ${this.name}!`);
    }
}
let person1 = new Person('Kim', 22);
let person2 = new Person('Lee', 23);

person1.sayHi();    // Hello, Kim!
```
위 예제에서 `function` 키워드를 이용해 `Person`이라는 생성자 함수를 만들었고, `new` 키워드로 `Person` 생성자를 받아 새로운 인스턴스를 생성하여 `person1`과 `person2`라는 객체를 만들었다. 생성자에는 위처럼 함수(메서드)를 넣을 수도 있다. 

### 2. `prototype`
JS는 흔히 프로토타입 기반 언어라고 불린다. 여기서 프로토타입이란 대체 무엇인가? 프로토타입은 JAVA나 C++과 같은 객체 지향 언어의 '상속'을 흉내낼 수 있게 해주는 일종의 메커니즘이다. 즉, 부모 객체의 속성(프로퍼티)와 함수(메서드)를 가져다 쓸 수 있게 해주는 '상속'을 구현하는 방식을 의미하며, 부모 역할을 해 주는 것이 바로 프로토타입이다.
